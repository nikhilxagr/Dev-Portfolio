# Day 3 Wireframes

## Goal

Lock the layout structure before the frontend setup begins.

This wireframe plan focuses on:

- clear hierarchy
- clean section flow
- strong first impression
- balance between full stack credibility and cyber security identity

## Visual Direction Summary

- layout style: bold, minimal, structured
- mood: modern developer portfolio with cyber security edge
- section rhythm: light green section backgrounds with dark blue cards and surfaces
- motion: subtle fades, slight slide-in, restrained hover feedback

## Desktop Layout Rule

- max content width: `1200px`
- section padding: `96px` top and bottom
- grid spacing: `24px` to `32px`
- cards should stay generous, not crowded

## Mobile Layout Rule

- single-column priority
- stacked sections with strong spacing
- stat cards and service cards should remain readable without tiny text

## Section Priority Order

The Home page should prioritize this order:

1. identity
2. proof
3. skills
4. projects
5. cyber security practicals
6. services
7. writing and thought leadership
8. contact

## Content Notes

- keep the hero strong and short
- do not overload the top of the page with too much text
- show proof early through stats and projects
- keep the services page simple and trustworthy
- avoid fake social proof
- testimonials section should stay hidden in version 1 because no real testimonials exist yet

## Day 3 Outcome

Day 3 is complete when:

- layout hierarchy is locked
- desktop and mobile direction are both clear
- section order is decided
- frontend implementation can start without guessing
