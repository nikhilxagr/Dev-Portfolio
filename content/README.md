# Content System

This folder keeps the portfolio content separate from the frontend and backend code.

The goal is simple:

- keep personal content easy to update
- reuse the same data in the frontend and backend
- avoid hardcoding text inside UI components

## Files

- `site.json`: main identity, hero copy, contact links, and global settings
- `page-copy.json`: short copy for each public page
- `stats.json`: coding and learning stats
- `projects.json`: featured work
- `practicals.json`: cyber security labs and writeups
- `services.json`: service offerings and pricing
- `blogs.json`: blog metadata and external links
- `testimonials.json`: real testimonials only

## Editing Rules

- Keep everything honest and easy to understand.
- Use real links, real work, and real testimonials only.
- Use `TBD` for missing values instead of guessing.
- Update the stats file manually once a week in version 1.
